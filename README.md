# EcoPilot AI - Autonomous Carbon Reduction Agent

**EcoPilot AI** is a production-ready, full-stack autonomous carbon footprint reduction agent. Built as a hackathon submission for the **Google Cloud Rapid Agent Hackathon (MongoDB Track)**, it goes beyond traditional Q&A chatbots by executing multi-step autonomous workflows to help users actively reduce their carbon output.

---

## Key Features

1. **Autonomous Footprint Agent**: Captures user commute, home energy, and dietary lifestyle statistics, processes standard emission formulas, writes data logs to MongoDB, and triggers model inference via Google Gemini to design custom plans.
2. **Interactive AI Coach**: Bottom-anchored chatbot utilizing active session state (carbon levels, active weekly milestones) to guide users through their green transition.
3. **Weekly Tasks Roadmap**: An organized 4-week calendar featuring category badges (Transportation, Home Energy, Lifestyle) and carbon impact markers (e.g., *Saves 14 kg CO₂*).
4. **Interactive Dashboard**: A responsive dashboard containing statistics cards and dynamic emissions breakdowns powered by Recharts.
5. **Session Isolation**: Utilizes local storage session tracking allowing easy onboarding, resetting, and demoing without requiring user login credentials.

---

## Tech Stack

*   **Frontend**: Next.js 16/15, TypeScript, Tailwind CSS (v4), Shadcn UI, Recharts, Lucide Icons
*   **Backend**: Next.js API Routes (Serverless endpoints)
*   **Database**: MongoDB Atlas (integrated using Mongoose ORM)
*   **AI Engine**: Google Gemini API via official `@google/genai` SDK
*   **Deployment**: Vercel-ready

## Database Schemas

### 1. `users`
Tracks session parameters and user responses:
```json
{
  "sessionId": "String (indexed)",
  "transportation": {
    "type": "String (Car/Bus/Train/Bike/Walk)",
    "dailyKm": "Number",
    "flightsPerYear": "Number"
  },
  "homeEnergy": {
    "electricityKwh": "Number",
    "cookingFuel": "String (LPG/Electric/None)"
  },
  "lifestyle": {
    "diet": "String (Vegan/Vegetarian/Non-Vegetarian)",
    "shopping": "String (Low/Medium/High)"
  },
  "goal": "String (Reduce by 10%/20%/30%)",
  "createdAt": "Date"
}
```

### 2. `carbon_logs`
Saves computed monthly footprint categories:
```json
{
  "sessionId": "String (indexed)",
  "transportationEmissions": "Number (dailyKm * 22 * 0.21)",
  "homeEmissions": "Number (electricityKwh * 0.82)",
  "lifestyleEmissions": "Number (dietFactor + shoppingFactor)",
  "totalEmissions": "Number",
  "month": "String",
  "createdAt": "Date"
}
```

### 3. `action_plans`
Saves action plans generated by Gemini:
```json
{
  "sessionId": "String (indexed)",
  "weeks": [
    { "week": "Number", "goal": "String", "impact": "String", "category": "String" }
  ],
  "geminiAnalysis": "String (AI summary overview)",
  "recommendations": ["String"],
  "createdAt": "Date"
}
```

## Setup Instructions

### Prerequisites
*   Node.js v18.17.0 or higher
*   NPM v9 or higher
*   A running MongoDB Atlas Cluster or local MongoDB instance
*   A Gemini API Key (obtainable from [Google AI Studio](https://aistudio.google.com/))

### Configuration
Create a `.env.local` file in the root directory and define the required parameters:
```bash
MONGODB_URI=mongodb+srv://<username>:<password>@cluster0.example.mongodb.net/ecopilot?retryWrites=true&w=majority
GEMINI_API_KEY=AIzaSyYourGeminiApiKeyHere
```

### Local Execution

1.  **Clone the workspace** and install dependencies:
    ```bash
    npm install --legacy-peer-deps
    ```
2.  **Start the local development server**:
    ```bash
    npm run dev
    ```
3.  Open [http://localhost:3000](http://localhost:3000) in your browser to test.

### Production Build
Validate compile correctness and build bundle:
```bash
npm run build
```

## MongoDB MCP Integration Note

This project is fully compatible with Model Context Protocol (MCP) servers. Utilizing a MongoDB MCP Server (e.g. `@smithery/mcs-mongodb` or standard mongo-mcp), agents can connect directly to your database endpoint (`MONGODB_URI`) to query and interact with user details, footprint metrics, and action plans.

---

### Sample MCP Commands:
*   **List Databases**: Check for `ecopilot` database.
*   **Query User Profile**: Find onboarding choices for a session:
    ```json
    {
      "collection": "users",
      "query": { "sessionId": "session_xxx" }
    }
    ```
*   **Generate Global Analytics**: Aggregate emissions averages across all sessions in the `carbon_logs` collection to gauge community impact..
